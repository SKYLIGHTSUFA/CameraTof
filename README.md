# Gige HLS Galaxy

Система захвата видео с камер Daheng Galaxy GigE и трансляции в HLS поток. 

Проект использует `harvesters` с GenTL Producer для получения кадров напрямую с камер и `ffmpeg` для кодирования видео (поддерживает аппаратное ускорение `h264_nvenc`). 
Также реализован дебайеринг Bayer-изображений на GPU с использованием `cv2.cuda` (при наличии OpenCV с CUDA) или на CPU как fallback.

## Особенности
- Захват сырых кадров с GigE камер с использованием Galaxy SDK (`GxGVTL.cti`).
- Высокопроизводительный демозаикинг (Debayering) на видеокарте Nvidia через `cv2.cuda`.
- Аппаратное кодирование H.264 через NVENC (`ffmpeg`).
- Формирование HLS-плейлиста и сегментов (секундные сегменты с заданным размером листа).
- Автоматическая трансляция с нескольких камер одновременно.
- Маппинг камер по их физическим/сетевым адресам.


## Требования
- ОС: Linux (x86_64)
- **Daheng Galaxy SDK** (GenTL драйвер). Указан в конфигурации: `/opt/galaxy_sdk/lib/x86_64/GxGVTL.cti`
- Установленный **FFmpeg** с поддержкой `h264_nvenc`
- (Опционально, но рекомендуется) **OpenCV** собранный с поддержкой **CUDA**.
- Python 3.8+ (для локального запуска).
- Для Docker: Docker Engine, Docker Compose, `nvidia-container-toolkit` (если используется GPU-версия).


## Установка и запуск

### Запуск через Docker
> **С использованием Nvidia GPU (NVENC/CUDA)**
```bash
docker-compose up -d
```
В этом случае поднимется контейнер, который прокинет внутрь все GPU ресурсы.

> **С использованием только CPU**
```bash
docker-compose -f docker-compose.cpu.yml up -d
```
*Убедитесь, что хостовая система и контейнер находятся в одной подсети (используется `network_mode: host`) для обнаружения GigE камер.*


## Конфигурация (`config.json`)

Основные параметры из `config.json`:
- `cti_file_path`: Путь к GenTL драйверу (CTI файлу). 
  Пример: `"/opt/galaxy_sdk/lib/x86_64/GxGVTL.cti"`
- `base_output_dir`: Папка, в которую сохраняются сгенерированные `.m3u8` и `.ts` файлы для раздачи по Web.
- `width`, `height`: Разрешение выходного видео после масштабирования (то, что идет в HLS).
- `cam_width`, `cam_height`: Разрешение ROI, устанавливаемое на камере.
- `fps_target`: Целевой FPS (будет передан в ffmpeg).
- `pixel_format`: Формат пикселей считываемых с камеры (например, `BayerRG8`).
- `gpu_device_ids`: Список ID видеокарт (например, `[0]`). Если `null`, система автоматически попытается обнаружить GPU через `nvidia-smi`.
- `use_nvenc`: Булево значение, разрешающее аппаратное кодирование на GPU.
- `camera_mapping`: Словарь для сопоставления серийных/сетевых имен камер Daheng читаемым ID, которые используются для формирования пути (например, `camera_{id}/index.m3u8`).

## Результат
Система формирует директории потоков вида:
```
hls_output/
├── camera_90.1.1/
│   ├── index.m3u8
│   ├── index0.ts
│   └── ...
├── camera_90.1.3/
│   ├── index.m3u8
│   └── ...
```
